The Clinical Problem Solvers framework teaches diagnostic reasoning through two reusable tools: illness scripts and diagnostic schemas. Both are designed to make expert reasoning transferable rather than case-specific.

An illness script is a structured mental summary of a disease with three parts: epidemiology and risk factors (who gets it), the time course and pathophysiology (how it behaves), and the key clinical features (how it presents, including the discriminating findings). Learners compare a patient's presentation against competing illness scripts to decide which best fits, and equally to decide what does not fit.

A diagnostic schema is a branching framework for a presenting problem (for example, a schema for anemia, hyponatremia, or acute kidney injury). The schema organizes the differential into a small number of categories with a logical first split, so the reasoner narrows the space systematically instead of pattern-matching to a single answer. Good teaching points name the relevant schema and the branch point that the current case turns on.

A common reasoning trap these tools guard against is premature closure: committing to the first plausible diagnosis without testing competing scripts or working the schema. Framing a teaching moment around the schema and the discriminating feature helps the learner generalize beyond the single case.
